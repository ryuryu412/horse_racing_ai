# 開発締めルーティン

「開発締め」と言われたら、このファイルを読んで以下を順番に実行する。

## 手順

1. **今日の作業内容をユーザーに確認**（会話の流れから読み取れる場合はスキップ）

2. **devlog.md に当日エントリを追記**
   - ファイル: `diary/devlog.md`
   - 形式: `## YYYY-MM-DD（曜日）`
   - セクション: 作業内容 / 実績（あれば） / 投票損益（あれば） / メモ

3. **README.md を更新**（変更があった機能・フローがあれば）

4. **メモリファイルを更新**
   - `memory/project_status_MMDD.md` を新規作成または更新
   - `memory/MEMORY.md` のインデックスを更新

5. **git commit & push**
   - 変更ファイルをすべてステージング
   - コミットメッセージ: `締め: YYYY-MM-DD 作業内容の一言まとめ`
   - push
