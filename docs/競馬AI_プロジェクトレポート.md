# 競馬AI 自作プロジェクト 詳細レポート

---

## 1. プロジェクト概要

### 1.1 何を作ったか

JRA（日本中央競馬会）のレースデータを使い、**機械学習によって毎週の競馬予想を自動化するシステム**を個人で開発・運用している。

単に予測するだけでなく、**出馬表CSVを渡すだけで予想PDFが自動生成される実用システム**として完成している。毎週の競馬開催日に実際に使用中。

### 1.2 システムの入出力

- **入力**: 出馬表形式のCSVファイル（JRA-VANから取得）
- **出力**: 横向きA4 PDF（印付き予想・各馬スコア・モデル統計を含む）

```
出馬表CSV → 特徴量生成 → 2モデル予測 → HTML生成 → Chrome自動PDF変換
```

### 1.3 開発者について

- プログラミング歴は長くない（開発初心者〜中級者）
- 機械学習・統計学の専門的な教育は受けていない
- Claude AI（Anthropic）との対話を活用しながら設計・実装・デバッグを継続
- 「動くものを作る」から始め、段階的に精度・品質を高めてきた

## 2. 開発の背景と目的

### 2.1 動機

競馬は「オッズ × 的中率」で期待値が決まる。JRAの控除率は約25%で、なにもしなければ長期的に75%しか返ってこない。
しかし**機械学習で真の勝率を推定できれば、オッズに割安馬を見つけられる**という仮説を立てた。

### 2.2 目的

1. 過去レースデータから馬の「本来の強さ」を数値化する
2. それをもとに各レースの出走馬をランキングする
3. オッズより強い馬（期待値プラス）を自動で特定する
4. 予想結果をわかりやすいPDFとして出力する

### 2.3 開発の基本方針

- **実用性重視**: 毎週使えるものでなければ意味がない
- **検証可能性**: ROI（回収率）を数値で確認できるようにする
- **自動化**: 手作業をなくし、CSV渡すだけで完結させる

## 3. データ

### 3.1 データソース

**JRA-VAN**（日本中央競馬会公式データサービス）から取得した過去レース全データ。

| 項目 | 内容 |
|------|------|
| 期間 | 約2005年〜2026年（約20年分） |
| 学習使用期間 | 2013年〜2025年末 |
| 検証期間 | 2026年1月〜現在 |
| 学習データ規模 | 約66万頭分（頭×レース単位） |
| 検証データ規模 | 約1万頭分（2026年） |

### 3.2 データの形式

1頭1レースを1行として管理。主な生データ列：

- **馬の情報**: 馬名・性別・年齢・馬体重・増減
- **レース情報**: 開催場所・距離・芝/ダート・クラス・頭数
- **騎手・調教師**: 名前・勝率
- **前走成績**: 着順・タイム・オッズ・人気
- **血統**: 父・母父など
- **過去走履歴**: 最大10走前まで

### 3.3 特徴量エンジニアリング

生データ272列に対して特徴量生成スクリプト（`01_make_features.py`）で以下を計算：

#### 主要な計算済み特徴量

| 特徴量名 | 内容 |
|----------|------|
| `騎手_勝率` | 騎手の通算単勝率 |
| `調教師_勝率` | 調教師の通算単勝率 |
| `近5走_平均着順` | 直近5レースの着順平均 |
| `前距離` | 前走の距離（今回との距離差を計算するため） |
| `間隔` | 前走からの経過週数 |
| `クラス_rank` | クラスを数値化（新馬=1, 未勝利=2, 1勝=3, 2勝=4, OP以上=5） |
| `性別_num` | 性別を数値化（牡=0, 牝=1, セ=2） |
| `キャリア` | 通算出走回数 |

#### 特徴量の総数

- モデルによって異なるが、1頭あたり最大**272列**の特徴量を使用
- 特徴量CSVのサイズ: 約1.3GB（parquet圧縮後: 219MB）

### 3.4 時系列リーク（look-ahead bias）への対策

**最重要の品質管理項目**。未来のデータが学習に混入すると、実運用で全く使えないモデルができてしまう。

対策:
- 学習データは**2025年12月31日まで**に完全限定
- 検証データは**2026年1月1日以降**のみ使用
- `_train_2025_step1_split.py` でデータ分割を明示的に実施
- 集計特徴量（騎手勝率など）に若干の混入リスクはあるが影響は軽微（2026年データは全体の1.7%）

## 4. AIモデルの設計

### 4.1 2段階のアーキテクチャ

**Step 1: LightGBM（勾配ブースティング・分類器）**

各馬の「単勝確率」を計算する。入力は272の特徴量、出力は0〜1の勝率スコア。

LightGBMを選んだ理由:
- 表形式データに対して非常に高精度
- 学習速度が速い
- 欠損値をそのまま扱える（競馬データには欠損が多い）

**Step 2: LambdaMART（学習ランキング / Learning to Rank）**

Step 1で計算したスコアをベースに、**レース内での順位を最適化**する。

通常の分類・回帰と異なり、「このレースの中でどの馬が1着に近いか」を直接学習する手法。同じレースの馬同士を正しく順序付けすることに特化している。

**→ 実際の予測では主にLambdaMARTのランキング（ランカー順位）を使用**

### 4.2 2種類のモデルを並走させる

**距離モデル（現行モデル）**
- モデルキー: `会場 × 距離`（例: `中山_ダ1200`）
- 例: 「中山ダート1200m専用モデル」
- コース適性を重視した特徴量で学習

**クラスモデル（サブモデル）**
- モデルキー: `芝ダ × 距離帯 × クラス`（例: `ダ_短距離_未勝利`）
- 例: 「ダート短距離・未勝利クラス専用モデル」
- クラス適性・クラス間の強さの差を重視した特徴量で学習

**なぜ2つ並走させるか**

1つのモデルだけでは見逃しが多い。距離の得意不得意と、クラスレベルの得意不得意は独立した要素なので、両方で評価することでより精度が上がる。

**両方でランカー1位を取った馬 → 本命候補**という考え方。

### 4.3 モデルの数

- 距離モデル: コース×距離の組み合わせ分（約21種類）
- クラスモデル: 芝ダ×距離帯×クラスの組み合わせ分（約21種類）
- 各モデルにLambdaMARTランカーも用意
- **合計100本以上の学習済みモデルファイル**（pickleで保存）

## 5. 指数・スコア体系

### 5.1 コース偏差値（絶対評価）

過去のコース全体を基準にした偏差値（50が平均）。

- `cur_コース偏差値`: 距離モデルによるコース基準偏差値
- `sub_コース偏差値`: クラスモデルによるコース基準偏差値

### 5.2 レース内偏差値（相対評価）

今日の出走馬だけで正規化した偏差値。同じレース内での相対的な強さを表す。

- `cur_レース内偏差値`: 距離モデルによるレース内偏差値
- `sub_レース内偏差値`: クラスモデルによるレース内偏差値

### 5.3 偏差値の差（穴馬シグナル）

```
偏差値の差 = レース内偏差値 − コース偏差値
```

この値がプラスに大きいほど、「このコース全体の平均より、このメンバー内では相対的に強い」ことを意味する。いわゆる**穴馬シグナル**。

- `cur_偏差値の差`: 距離モデル版
- `sub_偏差値の差`: クラスモデル版

### 5.4 ランカー順位（最重要シグナル）

LambdaMARTによるレース内順位予測。1位が最有力候補。

- `cur_ランカー順位`: 距離モデルのランカー順位
- `sub_ランカー順位`: クラスモデルのランカー順位

**ランカー順位が最重要**。偏差値の差はランカーの補助フィルターとして使う。

### 5.5 combo_gap（1位と2位の差）

```
combo_gap = (距離モデル1位と2位のスコア差) + (クラスモデル1位と2位のスコア差)
```

この値が大きいほど「1位の馬が突出している」ことを意味し、予測の信頼度が高い。

## 6. 印（マーク）システム

### 6.1 印の定義

予測結果に基づいて各馬に「印」をつける。4段階の印体系:

| 印 | 条件 | 意味 |
|----|------|------|
| **激熱** | 両モデルでランカー1位 & combo_gap≥15 & 偏差値差≥10 & オッズ3倍以上 | 最強の本命。突出して強い |
| **◎** | 両モデルでランカー1位 & combo_gap≥10 & オッズ3倍以上 | 本命 |
| **〇** | 両モデルでランカー1位 & combo_gap＜10 & オッズ3倍以上 | 本命（やや接戦） |
| **☆** | 片方が2〜3位・もう片方も3位以内 & オッズ5倍以上 | 穴馬候補 |

### 6.2 印の設計思想

- **両モデル一致が原則**: 片方のモデルだけ高評価でも印はつけない
- **オッズフィルター必須**: 1〜2倍台は配当が薄すぎて期待値がでないため除外
- **combo_gapで信頼度を分類**: 突出しているかどうかで激熱と◎を区別

### 6.3 週次の印の実績（2026年1〜3月）

毎週の開催で以下程度の印がつく（全レース合計）:

| 印 | 典型的な頭数/週 |
|----|---------------|
| 激熱 | 0〜3頭 |
| ◎ | 2〜6頭 |
| 〇 | 3〜8頭 |
| ☆ | 20〜35頭 |

## 7. 全自動予測パイプライン

### 7.1 実行コマンド

```bash
python src/06_predict_from_card.py data/raw/出馬表形式3月28日.csv
```

たったこれだけで全処理が完了する。

### 7.2 内部処理フロー

```
1. 出馬表CSVの読み込み・変換
   └ 列名の正規化（複数フォーマットに対応）
   └ 日付パース・開催コード生成

2. オッズ補完
   └ CSVにオッズがあればそのまま使用
   └ なければYahoo競馬から自動スクレイピング

3. 特徴量生成
   └ card_extra.csvへ書き込み
   └ 01_make_features.py を自動実行
   └ parquetキャッシュを自動更新

4. モデルプリロード
   └ 当日出走コースに対応するモデルのみ選択・ロード
   └ 距離モデル + クラスモデル + 各ランカーを一括読み込み

5. レース別予測
   └ 各レースをグループ化
   └ 両モデルで predict_proba → 偏差値計算 → ランカー順位計算

6. 印付け
   └ 全馬の指数を基に激熱・◎・〇・☆を判定

7. HTML生成
   └ 発走時刻順にソート
   └ サマリーページ + レース別詳細ページ
   └ データ品質アラートボックス

8. PDF自動生成
   └ Chrome ヘッドレスモードで HTML → PDF変換
   └ Googleドライブに保存
```

### 7.3 処理時間の目安

| 処理 | 時間 |
|------|------|
| Parquet読み込み | 約2秒 |
| 特徴量生成（01_make_features.py） | 数分 |
| モデルプリロード | 約3〜5秒 |
| 全レース予測 | 数秒 |
| HTML/PDF生成 | 数秒 |
| **合計** | **数分〜10分程度** |

## 9. 検証手法と結果

### 9.1 検証の設計（Out-of-Sample検証）

最も重要なのは「学習に使っていないデータでの性能」。

- 学習データ: 2013〜2025年（約66万頭）
- 検証データ: 2026年1月〜現在（約1万頭）
- データは時系列で完全分離。シャッフルなし。

これにより「2026年のリアルな競馬で、このAIはどの程度通用するか」を正確に測定できる。

### 9.2 ROI検証結果（2026年1〜3月）

| 戦略条件 | 頭数 | 単勝ROI |
|----------|------|---------|
| 両Rnk=1（オッズ制限なし） | 339頭 | +21.4% |
| 両Rnk=1 & オッズ3倍以上 | 169頭 | +52.7% |
| **両Rnk=1 & odds≥3 & 偏差値差≥10** | **62頭** | **+127.3%** |

## 10. データ品質管理

### 10.1 自動アラートシステム

予想PDF内に「データ品質アラート」ボックスを自動表示する機能を実装。
**最終的な判断は人間が行う**ハイブリッド運用を実現。

#### 検出する3種類のアラート

**① 前走地方競馬・海外帰り**

parquetデータはJRA（中央競馬）のみ収録。地方競馬（大井・川崎・園田 等）や海外帰りの馬は、前走データが反映されない。

```
検出方法: 出馬表の「前場所」列が国内JRA場所リスト外かどうか
表示例: 「前走地方競馬『大井』（前走データ未反映の可能性）」
       「前場所『UAE』が国内場所リスト外（海外帰り疑い）」
```

**② 間隔ズレ**

出馬表に記載の前走からの間隔と、parquet計算値が10週以上乖離している場合。地方・海外帰りの検出の補助として機能。

```
検出方法: abs(出馬表の間隔 - parquet計算間隔) > 10週
表示例: 「間隔: 出馬表4週 vs 算出28週（差24週）」
```

**③ 重要特徴量の欠損**

キャリア5走以上あるにも関わらず「近5走平均着順」がNaNの場合、特徴量生成に問題がある可能性。

```
検出方法: 近5走_平均着順 is NaN & キャリア > 5
表示例: 「近5走_平均着順が欠損（キャリア8走）」
```

### 10.2 parquetデータの特性と限界

- 収録範囲: JRA中央競馬のみ
- 地方競馬（NAR）・海外レースは含まない
- 「各馬の最新行」を使って当日出走馬の特徴量を補完する方式
- 連続して翌日出走するような馬は稀なため、「最新行に翌日以降のデータが混入する」リスクは極めて低い

## 付録: 主な技術用語

| 用語 | 説明 |
|------|------|
| LightGBM | Microsoftが開発した高速・高精度な勾配ブースティング機械学習ライブラリ |
| LambdaMART | 検索エンジン等で使われる学習ランキング手法。レース内順位予測に応用 |
| Out-of-sample検証 | 学習に使っていない未来のデータで性能を測定すること。最も信頼性の高い評価方法 |
| ROI（回収率） | 投資した金額に対してどれだけ回収できたかの割合。100%が収支ゼロ、127%なら27%の利益 |
| 偏差値の差 | コース全体平均との差。プラスが大きいほど「このメンバー内では実力以上に見える馬」 |
| ランカー順位 | LambdaMARTによるレース内順位予測。1位が最有力候補 |
| combo_gap | 距離モデルと クラスモデルそれぞれの「1位と2位のスコア差」の合計。大きいほど1位が突出 |
| parquet | CSVより高速に読み書きできる列指向データフォーマット。CSVの1/6のサイズで2秒で読める |
